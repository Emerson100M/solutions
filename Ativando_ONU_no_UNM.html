Autorizando ONU no UNM

1 - ONU FIBERHOME

Identificando e registrando a ONU

01 seleciona a OLT
02 obtain unautorized ONU —> all
03 add to the ONU Authority List
04 As “physical ID”
05 current Row —> create on device
06 current Row —> Save to database
07 procura o serial da ONU autorizada
08 seleciona a ONU —> Service config

::: ONU em modo ROUTE :::

09 Wan service —> add —> 1
10 Wan mode —> internet
11 Wan connection —> route
12 Vlan —> adiciona a VLAN do local
13 Vlan cos —> 4
14 Nat —> enable
15 Wan dsp —> PPPOE
16 Wan IP —> IPV4 e IPV6
17 IPV6 —> DHCPV6
18 WAN —> username e password (PPPOE do cliente)
19 habilitar as checkbox conforme o modelo da ONU

  • modelo 02F (AN5506-02-F) —> (LAN_1-2) + (2.4G_SSID_1)
  • modelo 04F (AN5506-04-F) —> (LAN_1-2-3-4) + (2.4G_SSID_1)
  • modelo HG e AX (HG6145F) —> (LAN_1-2-3-4) + (10G_LAN) + (2.4G_SSID_1) + (5G_SSID_1)

20 create on device
21 save to database

::: ONU em modo BRIDGE :::

09 Port service config —> data port —> LAN1
10 Service configuration —> ADD ou MODIFY
11 CVLAN ID ( adiciona a VLAN do local)
12 CVLAN MODE —> TAG
13 Vlan cos —> 4
14 OK
15 create on device
16 save to database

Observações e outros recursos do UNM

  • Para capitão poço e ourém —> ONU ATTACHE —> QUINQ (observar SLOT e PON)

—> se PON >8 (inserir: QINQvlan_slotB)
—> se PON <8 (inserir: QINQvlan_slot)

  • USER DHCP SERVER PARAMETER —> troca a faixa de dhcp (ex: 192.168.1.1)

  • ONU DEAUTORIZATION —> desautoriza a ONU (fazer isso em trocas de endereço ou trocas de equipamento)

  • ONU RESET —> reinicia a ONU

  • OPT MODULE —> verifica o sinal ótico da ONU

  • ONU LOCAL MANAGE —> config ENABLE, ani ENABLE, web service 8021 ou 8080 (ativa acesso remoto http)

2 - ONU HUAWEI

Identificando e registrando a ONU

12 Service profile —> veip huawei

14 save to database
